# 物理教材用ドローソフト 要件定義書（プラグインアーキテクチャ版）

# 1. コンセプト

本ソフトは、高校・大学初級レベルの物理教材を効率よく作成するためのブラウザベースのドローソフトである。

一般的なドローソフトではなく、「物理オブジェクト」を配置・編集することに特化したエディタとする。

また、本ソフトは長期的な拡張を前提とし、**すべての物理オブジェクトをプラグインとして実装できるアーキテクチャ**を採用する。

# 3. 設計思想

アプリ本体は

* ワークスペース
* キャンバス
* 描画
* 選択
* Undo/Redo
* 保存
* 出力

のみを担当する。

物理図形の知識は本体に持たせない。

すべての物理図形はプラグインとして提供する。

---

# 4. Physics Object Plugin

すべてのオブジェクトは共通インターフェースを実装する。

プラグインは最低限、以下を提供する。

* オブジェクト名
* カテゴリ
* アイコン
* 初期サイズ
* SVG描画
* プロパティ
* JSON保存
* JSON読込
* 選択表示
* バウンディングボックス
* 回転
* 拡大縮小
* スナップ位置

アプリ本体はオブジェクトの種類を意識しない。

新しいオブジェクトは登録するだけでツールボックスへ追加される。

# 15. JSON

すべての図はオブジェクト配列として保存する。

各オブジェクトは

* pluginId
* version
* property

のみを持つ。

描画方法は保存しない。

プラグインがJSONから再構築する。

これにより、新しいプラグイン追加時も保存形式を大きく変更せずに済む。

---

# 16. 拡張性

新しい物理オブジェクトを追加する際は、

1. プラグインを作成
2. 登録

のみで利用可能になること。

既存コードの修正は不要とする。

これは本ソフトの最重要要件である。

---

# 17. 非機能要件

* 軽快な動作
* オフライン利用
* ブラウザのみで完結
* SVG品質を重視
* 教員が直感的に利用できるUI
* 将来的に100種類以上のプラグイン追加を想定した保守性を確保する

---

# 18. 開発方針

MVPでは以下の標準プラグインを実装する。

* 線
* 円
* 長方形
* 矢印
* テキスト
* LaTeX
* ブロック
* バネ
* ベクトル
* 力ベクトル

アプリ本体はプラグイン管理機構を完成させることを最優先とする。

以降の機能追加は、可能な限りプラグインの追加だけで実現する。
